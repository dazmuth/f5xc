Lab 15: Creating Routing Configuration

Estimated time for completion: 35-45 minutes

Objectives

At the end of this lab students will be able to:
  • Edit an HTTP(s) Load Balancer
  • Demonstrate the creation of a simple and redirect rerouting configuration
  • Demonstrate the various Route Types
  • Configure a Direct response page
  • Re-order routing configuration rules
        
      
Step
Edit HTTP(s) Load Balancer
  • In upper-left corner click Select service, then type (or click) Multi-Cloud App Connect
      
    Verify that your namespace is selected.
  • In the Manage section, hover over Load Balancers then select HTTP Load Balancers
  • Locate studentX-https-lb, scroll to the right, and click the 3 dots on the right
  • Click Manage Configuration
  • Click Edit Configuration
  • Scroll down to the Routes section and click Configure
  • Click Add Item
Step
Review Route Types
  • Click the drop-down for Route Type and review the four types: Simple, Redirect, Direct Response, Custom Route Object
  • Simple Route sends traffic to specific origin pools. Selecting Simple Route reveals fields Headers, Origin Pools, and Host Rewrite method
    • Use case for using Headers: your application has recurring issues with a browser like Firefox. Use headers to force all requests from that browser into a specific pool for analysis
    • Use case for using Origin Pools: a company has different portals for its application where each portal is handled by separate teams. For example, a medical insurance company may have one portal for providers, and another for benefits
    • Use case for using Select Host Rewrite Method: usually the Automatic method is sufficient, but suppose you use a FQDN to identify a specific cloud provider resource, such as a load balancer, and you want to ensure that requests are proxied correctly
      While not in scope for this course, F5XC provides the ability to create highly-specific matching conditions to manage routes; for example, to reroute URLs that contain query parameters (e.g. filters/indices built right into the URL like https://testsite.com/articles?sort=ASC&page=4 ). To do this go to Multi-Cloud App Connect > Manage section > Virtual Hosts > Add Route > under List of routes section, click Configure > Add Item.

      For more information see https://docs.cloud.f5.com/docs/
  • Redirect Route redirects traffic from your application to any URL
  • Direct Response Route displays a response page. Perhaps a product is temporarily out of stock, so rather than rewriting that application's page, a quick response page is displayed instead
  • Custom Route Object
    https://f5cloud.zendesk.com/hc/en-us/articles/4405130078103-How-to-setup-path-based-routing-or-application-load-balancing
    This advanced method is used to forward request with complicated parameters. A use case is to set up a load balancer to forward requests to different origin servers depending on the language chose by the website visitor.

    Usage of Custom Route Object first requires the creation of a route, which is accomplished in Multi-Cloud App Connect > Manage > Virtual Host > Add Route
Step
Select Redirect Route
  • Route Type: Redirect Route
  • HTTP Method: GET
Step
Review Path Match Types
  • Click the drop-down for Path Match and review the three types
    • Prefix: this matches on anything after the hostname
    • Path: must be an exact path to match. Do not use wildcards or asterisks. The path starts after the hostname
    • Regex: use standard regular expression matches, starting after the hostname. For help with regex, see https://regex101.com
Step
Select Path Type
  • Path match: Path
      
    In this scenario all requests for the Salt & Pepper shaker must be redirected to instead display the Bamboo Glass Jar.
  • In the Path field, type (or copy and paste) the path of the Salt & Pepper Shakers: /product/LS4PSXUNUM
Watch out for leading or trailing spaces when copying/pasting into all fields.
Step
Review Redirect Parameters
  • In the Redirect Parameters section, click the drop-down for Protocol and review its options
    • incoming-proto preserves the incoming protocol (HTTP or HTTPS) and redirects using the same protocol
    • http redirects the request as http (since most websites automatically resolve to https, this option is rarely selected)
    • https redirects the request as https (most commonly-selected option)
Step
Select HTTPS Redirect Parameter
  • Protocol: https
  • Host: type studentX.aws.learnf5.cloud
      
    This is the hostname of the site, with or without www. Do not include the trailing slash (/)
  • Redirect Path: Path
  • Path: type (or copy and paste) the path for the Bamboo Glass Jar: /product/9SIQT8TOJO
  • Response Code: type or select 302
    301 (permanently moved) is the default value, but 302 (temporarily moved) is also accepted in the logs. Note that any value can be entered here, but only 301 & 302 persist through to the logs.
  • Query Parameters: select the default value of Retain All Parameters
      
    In production, if the URL is dynamic and contains index or page information, it may make sense to instead select Remove All Parameters
  • Click Apply
      
    The new Redirect route is displayed on the Routes page.
Step
Create Direct Response Route for pages containing "V" in its SKU
In the remaining examples, two different pages contain items with a V in its SKU, but we want to ensure that they are displayed in a particular order using two different configurations.
  • Click Add Item
  • Route Type: Direct Response Route
  • HTTP Method: GET
  • Path Match: Regex
  • Regex: .*V.*
      
    In other words, type the following: period, asterisk, capital V, period, asterisk.
       
    This ensures that it will match any path values before a capital V, and any values afterwards.
    Notice that leading and trailing slashes are not required in this field. Do not enter slashes or apostrophes or other delimiters at the beginning/end, because not only does F5XC not require them, but F5XC will return no values if used.
  • In the Direct Response section, click Configure
Step
Configure Direct Response
  • Response Code: 200
      
    The 200 code means "OK" but in production other values may be entered as desired.
  • Response Body: type This V site is down
      
    In production, HTML may be entered as desired. For training this message will help us distinguish between responses.
  • Click Apply
  • Click Apply again
      
    The new Direct Response route is displayed on the Routes page.
Step
Create Direct Response to indicate Tank Tops are Out of Stock
  • Click Add Item
  • Route Type: Direct Response Route
  • HTTP Method: GET
  • Path Match: Prefix
  • Prefix: /product/66
      
    This will match any product page that begins with 66.
    Do not enter an asterisk, as asterisks only work with Regex.
  • Direct Response: click Configure
Step
Configure Direct Response Route
  • Response Code: 200
  • Response Body: tank tops are temporarily out of stock
  • Click Apply
  • Click Apply again
      
    The new Direct Response route is displayed on the Routes page.
  • Click Apply
  • Scroll to the bottom of the page and click Save and Exit

Q: What happened? A direct response route was created for tank tops to display a page with the text "Tank Tops are temporarily out of stock."

A: Look at the URL for Tank Top. It begins with 66 then has a V. This therefore triggers the second rerouting rule, which is looking for any product URL containing a V. The routes must be reordered in F5XC to ensure they are triggered in the desired order.
Step
Reorder Routes in F5XC
  • Toggle back to F5XC, which is displaying the HTTP Load Balancers screen
  • Locate studentX-http-lb, scroll to the right, and click the 3 dots on the right
  • Click Manage Configuration
  • Click Edit Configuration
  • Scroll down to the Routes section and click Edit Configuration
  • Hover over the 6 dots to the left of the Route for /product/66 at the bottom of the screen, then click and drag it above the second route
  • Click Apply
  • Scroll to the bottom of the page and click Save and Exit
Step
Retest Tank Top Routing
  • Return to home page of studentX.aws.learnf5.cloud
  • Refresh the browser for studentX.aws.learnf5.cloud
  • Click Tank Top
      
    The tank tops are temporarily out of stock page is displayed.
Step
View rerouted requests in dashboard

    Verify that your namespace is selected.
  • In the Virtual Hosts section, click HTTP Load Balancers
  • Locate studentX-https-lb and click the Performance Monitoring Link underneath the name navigate to Requests
  • Navigate to Requests
  • Within the Request & Response section, notice req_path and rsp_code_details
