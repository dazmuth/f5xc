Lab 4: Creating Cloud Site

Estimated time for completion: 25-35 minutes

Objectives

At the end of this lab students will be able to:
  • Create an AWS VPC site in F5XC and apply it to AWS instantiation
  • See how Instances are created in AWS from F5XC
        
      
Step
Add an AWS VPC Site
  
Add the client Virtual Private Cloud (VPC) in Distributed Cloud Services. This assumes that AWS cloud credentials have already been created (see previous lab).
  • In upper-left corner click Select service, then click Multi-Cloud Network Connect
  • In the Manage section, hover over Site Management, then click AWS VPC Sites
  • Click Add AWS VPC Site
  • Name: studentX-vpc
Step
Create Labels
  • For Labels, click Add label then select two values:
    • studentX-key
    • studentX-value
    This Key Pair links the virtual site to the VPC, causing the pods to be built in that virtual site.
  • Click anywhere outside of the Labels area to escape
      
    As usual, a Description is optional.
Step
Select Site Type and VPC Parameters
  • In the Site Type Selection section, under AWS Region, type or select us-east-1
  • Verify that the VPC drop-down displays New VPC Parameters
    If Existing VPC ID is selected, the VPC must match exactly what is displayed in AWS, which will include a system-generated series of digits
    Click the VPC drop-down to show both options, then inform class that if Existing VPC ID is selected, the VPC must match exactly in AWS. Open AWS, go to Your VPCs, and show an example.
  • Verify that the AWS VPC Name drop-down displays Autogenerate VPC Name
  • Under Primary IPv4 CIDR block, type: 172.31.0.0/16
     
    All Subnets used in this VPC must be part of this CIDR (Classless Inter-Domain Routing)
    Hover over the ? icon to show the requirements for this block.
Step
Select Ingress/Egress
https://docs.cloud.f5.com/docs/how-to-site-management/create-aws-site
  
View information at this link for details on more complicated interface deployments.
  • Verify that the drop-down for Select Ingress Gateway or Ingress/Egress Gateway displays Ingress Gateway (One Interface)
  • Under Ingress Gateway (One Interface), click Configure
  • Do not modify the value for the drop-down for AWS Certified Hardware
  • In the Ingress Gateway (One Interface) Nodes in AZ section, click Add Item
  • In the AWS AZ Name drop-down, type us-east-1a or click See Suggestions and select us-east-1a
    https://aws.amazon.com/about-aws/global-infrastructure/regions_az/
     
    The selected Availability Zone becomes the default zone, but AWS may shift traffic to one of the other subzones to maximize application performance.

Step
Allow HTTP and HTTPS Port
  • Click the drop-down for Allowed VIP Port Configuration and select Allow HTTP & HTTPS Port
  • Click Apply
Step
Define Deployment parameters
  • In the Cloud Credentials drop-down field, select system/aws-learnf5
      
    This value is defined when configuring Cloud Credentials.
  • Scroll down to Site Node Parameters section
Step
Define Site Node Parameters
  • In the Site Node Parameters section, under AWS Instance Type for Node, select or type m5.4xlarge
      
    The instance type is based on desired performance. The larger the instance, the better the performance, but of course AWS charges more for larger instances. Work with your Development team to determine the best AWS instance size for your application.
    In training be sure to select m5.4xlarge. Otherwise the application will not perform correctly.

    • For the to-be-destroyed VPC, click the 3 dots under Actions and select Clone Object
         
      It is not necessary to wait until the old VPC is fully destroyed to Clone Object.
    • Provide a name that differs from the previous VPC for this newly cloned VPC
    • In the Site Node Parameters section, select the correct AWS Instance Type for Node
    • Verify all other settings
    • Click Save and Exit
    • Click Apply to begin instantiation of cloned VPC

Step
Allow DNS and SSH services
  • In the Advanced Configuration section, click the slider for Show Advanced Fields
    The AWS Tags field should allow the assigning of labels to VPC deployments, which then appear in AWS. But as of June 2023, this feature isn't working as expected.
  • In the drop-down for Services to be blocked on site, select Allow access to DNS, SSH services on Site
  • Click Save and Exit
Step
Instantiate the Instance
  • Find studentX-vpc in the list and click Apply
  • In training, the instructor will display AWS. In a production environment, access AWS and watch the instance being created
In production instantiation usually takes 15-25 minutes. But in the training environment, with multiple Instances instantiating at once, this may take up to 30 minutes.

Once the Site Admin State is Online and the Status is Applied, the Instance is built.

https://docs.cloud.f5.com/docs/how-to/site-management/create-aws-site
Toggle to AWS and select Instances. Notice that the VPCs are being constructed, with each named "master-0." Once its Instance State is Running, check F5XC and verify that its status is Applied. For each instance, perform the following:
  • Click the checkbox for that instance to display details below (ensure the Details tab is displayed)
  • Locate the VPC ID in the details, which contains the student ID (studentX)
  • Click the little edit box to the right of each instance to rename it (since every instance by default is named "master-0")
  • Rename it to "studentX"
Step
View VPC Actions
  • For studentX-vpc, click the 3 dots to the right under Action
      
    Use these options to view finer details about the VPC deployment
  • Click Terraform Parameters
  • Click the Apply Status tab
      
    Details about the instantiation are displayed.

Expected Results

Students have:
  • Created an AWS VPC site in F5XC and applied it to AWS for instantiation
  • Seen the various AWS region types and when to use them
  • Seen how Instances are created in AWS from F5XC
        
      

Review

  • Creating a VPC in the F5XC GUI builds a corresponding VPC in AWS with a Private DNS address
  • Cloud VPCs require Key Pairs associated to a virtual site
  • AWS Instance size is based on application performance requirements, which is a combination of expected traffic, and complexity of application itself
  • When adding Public key SSH info, the corresponding Private key must exist inside the .ssh folder as id_rsa (in training this is home/student/.ssh)
