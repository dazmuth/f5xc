Demonstration 1: Introduction to F5 Distributed Cloud Console

Estimated time for completion: 25-30 minutes
Accounts are already setup for training. The instructor will demonstrate these steps; following are the steps for your records.
  
Link to User Management official documentation: https://docs.cloud.f5.com/docs/how-to/user-mgmt
This demo lab is intended to showcase the GUI. Students can follow along, but this is really more of a lecture than a lab. They should be watching you and learning, and not blindly following the steps.

Objectives

At the end of this lab students will be able to:
  • Know how to create new users and roles in F5XC
  • View common F5XC Use Cases
  • Access Support documentation
  • View Account Settings
  • View key Administration Settings
        
      
The instructor provides the student number for each student. Throughout the labs, "studentX" is used to sign in and for assigning a variety of objects. For every instance, replace the "X" with your student number.

For example, for student 3, use student3
Student usernames (which are just email addresses) are student1@f5.com through student9@f5.com. Passwords should change before each class.

For information on Student and Instructor Login information, view the Instructor Guide (Course Setup > Before Class Process Steps)

Explore GUI

These options differ based on your role and account settings.
Step
View home page options
  • Within the F5 Distributed Cloud GUI, if the home page is not displayed, click the F5 logo in upper-left corner
     
    The most commonly-used services are displayed at the top of the page. These options differ based on your role.
    If any of the Services displays an option for "Preview" or "Private Preview" click it to view an About page which provides Technical Documentation and instructions on how to activate this feature.
  • Click any of these Service panes to view details, then click the F5 logo in upper-left corner to return to the home page
Step
View Services
  • Scroll midway down and click the drop-down for All Services
      
    This displays a break-down of each of the service panes listed above in addition to options that may not be displayed above.
  • Scroll through the list of Services to see the various options
Step
View Use Cases to Build a Solution
  • Scroll to bottom of page to the Build a Solution section
     
    This section displays a few examples of use cases taken from the Teachable website. Use this section to get a high-level example of how F5XC can be used.
Step
View Floating Help
   
This "DAP" (Digital Access Platform) feature provides resource material within the GUI so users don't have to navigate away from the tool for basic help.
  • Along the scroll bar on right-hand side, click the floating question mark
  • In the Help tab, click New Features
      
    This is a direct link to the F5XC Change logs
  • Click New Features again to minimize its options
  • Click Resources and/or Guides
      
    Resources     and Guides are links to docs.cloud.f5.com
  • Click the Onboarding tab
      
    These options provide a guided tour around the GUI
  • Click the X in Floating Help "How Can we Help You?" window to close it
Briefly discuss the other options listed here, and click a few as needed to demonstrate how they guide you through the GUI.
Step
Access Support links
  • Scroll to top of page and click the drop-down for Support in the upper-right corner
      
    Help and Support is offered on every page of the application under the Support drop-down.
  • Click Contact Support
  • Click the Service drop-down to display options
  • Click the Type drop-down then select options in the other drop-downs to explore topics
      
    This is how to open a support ticket. After filling out this form and clicking Submit request, a support engineer will contact you within an hour or so to help with your problem.
    Do not create a Support Ticket during class.
  • Click Cancel
  • Click Support in the upper-right corner again
        
    These other options are self-explanatory ways to get help or learn about the product.
Briefly discuss the other options listed here, and click a few as needed to demonstrate them.
Step
View Account Settings
  • Click your avatar in upper-right corner
  • Select Account Settings
      
    In production the editability of these options may be different for you. To edit the displayed GUI options, view Work domains and skill level.
  • Within Work domains and skill level click Edit work domains & skills
  • Verify that Super User is checked
      
    These options drive which options are displayed by default in the GUI. Since XC is designed as a collaborative tool between DevOps/NetOps/SecOps/Developers, this section allows the system administrator to customize views so that users aren't inundated with options that don't matter to them.
  • Click Cancel to exit
Notice the breadcrumb across the top of the screen. It is not clickable but shows where you are in the GUI at all times.

Review key Administration Features

Click through each of the options listed in Administration and discuss key points of each feature
On every page underneath the F5 logo in the upper-left corner is the option to Select service. Clicking it exposes the common services as defined by the account settings, and also displays the Quick Search field. Select a service or just type what you want in the field.
Step
View Tenant Settings
  • In the Tenant Settings section, click Tenant Overview and note the following features
    • Add your own logo in Custom Logo section
    • In Credential Expiry Policy section, define expiration periods for API Certs and KubeConfig Certs, and API Tokens
    • Define Tenant Configuration options to modify password policy settings
    • In Support Access, modify read/write access to tenant
  • In the Tenant Settings section, click Login Options
    • Enable two factor authentication, or set up SSO authentication using an OpenID Connect (OIDC) provider
  • In the Tenant Settings section, click Public IP Addresses
    • Click "Add Public IP" to create a ticket with the F5XC Support team to define public IP Addresses for this tenant
  • In the Tenant Settings section, click Quotas
    • Scroll through the list to view the default object quota values and current usage
    • To change quotas, open a Support Ticket
Step
View Personal Management
 
The My Account options were discussed in previous steps, and My Namespaces is discussed in the "Create Namespace" lab.
  • In the Personal Management section, click Credentials
    • Use to create API Certificates, API Tokens, and vK8s KubeConfig
    vK8s KubeConfig credentials are created in a later lab but can also be created manually here
Step
View IAM
  • A valid email address is required in order to receive the activation email from F5XC, but that email does not need to be an email address tied to a corporate account. It can be a Yahoo account, etc.
  • Note that specific roles and namespaces may be assigned to a user using the option Assign Roles and Namespaces
  • Click Cancel when finished demonstrating Add New User features
  • In the IAM section, click Users
    Do not create a new User in class since they are already created for training, but in production, create new users by clicking Add User
    • A valid email address is required in order to receive the activation email from F5XC, but that email does not need to be an email address tied to a corporate account. It can be a Gmail or Proton or Yahoo account, etc.
    • Note that specific roles and namespaces may be assigned to a user using the option Assign Roles and Namespaces
    Do not create a User during the training class. The instructor demonstrates the options for reference only.
  • In the IAM section, click Groups
    https://docs.cloud.f5.com/docs/how-to/user-mgmt/users#create-user-groups-and-add-users
    • Groups may be helpful with dozens or hundreds of users. Click the link the link above (or the Tech Docs link on the Add Group page) for details
    Two groups are present in the training tenant: managed-admin and managed-monitor. These are used with the Delegated Access service to allow the F5XC training developers the ability to manage both the development tenant and this customer tenant. In production, you likely have only one tenant, but if you manage multiple tenants these groups are necessary to allow specific users the ability to manage multiple tenants.
  • In the IAM section, click Roles
    • Roles are used to further define F5XC access; for example, billing or monitoring users do not need the same access as developers
    • Role names that begin with "f5xc" or "ves-io" are default roles that may be edited but not deleted
  • In the IAM section, click Delegated Access
    https://docs.cloud.f5.com/docs/how-to/user-mgmt/mda
    • Tenant Access is used by the Delegated Access Service. In production this is often used to allow third parties (such as service consultants) to access your tenant in order to manage services and configurations. Click the link the link above (or the Tech Docs link on the Add Allowed Tenant page) for details
  • In the IAM section, hover over RBAC Policies, then select Built-in Policy Rules
    • This view-only screen displays default RBAC Policies
  • In the IAM section, click API Groups
    • This view-only screen displays the various API Groups (also displayed in the Roles section)
  • In the IAM section, click Service Credentials
    https://docs.cloud.f5.com/docs/how-to/user-mgmt/credentials#service-credentials
    • The same as when creating Credentials as displayed in the Personal Management section, except this allows creating credentials for a specific email address. Click the link the link above (or the Tech Docs link on the Add Credential page) for details
Step
View Support
These Support options are more detailed than those provided by clicking Support to the right of your avatar (as described above).
  • In the Support section, click Get Help
    • This page displays the various methods to receive F5XC support
  • In the Support section, click Requests
    • This page displays every Service Ticket opened for this Tenant
