Lab 5: Creating and Sending Alerts

Estimated time for completion: 35-40 minutes

Objectives

At the end of this lab students will be able to:
  • Deploy Alert Policies and Alert Receivers at the Tenant and Namespace level
  • View Alerts in the F5XC console
  • Review Alerts in email
        
      
Throughout class, alerts are triggered as Labs are created and traffic increases.
Alerts can be built at the Tenant or Namespace level. Alerts built at the Namespace level can not be applied at the Tenant level.
https://docs.cloud.f5.com/docs/ves-concepts/monitoring#alerts
https://docs.cloud.f5.com/docs/reference/alert-ref
  
Step
Create Tenant-Level Alert Receiver
  • In upper-left corner click Select Service, then type (or click) Shared Configuration
  • Expand the Manage section, hover over Alerts Management then select Alert Receivers
  • Click Add Alert Receiver
  • Name: studentX-tenant-alert-receiver
  • In the Description field, type studentX tenant alert receiver
    A description is optional in production, but used here in training to distinguish objects created in later steps.
  • Click the drop-down for Receiver and select Email
      
    https://docs.cloud.f5.com/docs/how-to/alerting

    For additional information on the available Receivers, click the above link.
  • Email: studentX@f5.com
  • Click Save and Exit
Step
Verify Email for Tenant-Level Alert Receiver
  • Verify your window displays Shared Configuration > Manage > Alerts Management > Alert Receivers
  • In the Manage section, Click the 3 dots for studentX-tenant-alert-receiver and select Verify Email
  • Click Send email
         
    The email address must be verified before F5XC is able to send alerts to it. The Send Test Alert option does not work until the email has been confirmed
  • In production, open the corresponding email inbox and copy the six digit verification code. In training, the Instructor will display these emails and provide the verification code
  • Click the 3 dots for studentX-tenant-alert-receiver and select Enter verification code
  • Paste the six digit verification code. If applicable, be sure any leading zeros are included
Step
Test Tenant-Level Alert Receiver
  • Verify your window displays Shared Configuration > Manage > Alerts Management > Alert Receivers
  • Click the 3 dots for studentX-tenant-alert-receiver and select Send Test Alert
  • Click Send test alert
  • In production, open the corresponding email inbox to view test the alert. In training, the Instructor will display these emails
Step
Configure Tenant-Level Alert Policy
  • Verify your window displays Shared Configuration > Manage > Alerts Management > Alert Receivers
  • In the Manage section, hover over Alerts Management then select Alert Policies
  • Click Add Alert Policy
  • Name: studentX-tenant-alert-policy
  • In the Description field, type studentX tenant alert policy
    A description is optional in production, but used here in training to distinguish objects created in later steps.
  • In the Alert Receiver Configuration section, click the drop-down for Alert Receivers
  • Select shared/studentX-tenant-alert-receiver
  • On the far-right side click the slider for Show Advanced Fields
  • In the Receiver Default Notification Parameters section, click Configure
  • Change value for Notify Interval For a Alert to 30m
    All fields must be in 2-digit number form (e.g. 60m, 60s)
  • In the Notification Grouping section, verify that the Wait to Notify value is 30s
  • In the Notification Grouping section, change Notify Interval for a Group to 30s
  • Click Apply
Step
Configure Tenant-Level Alert Policy Rules
  • Verify your window displays Shared Configuration > Manage > Alerts Management > Alert Policies > [Add Alert Policy] > Metadata
  • In the Policy Rules section, click Configure
  • Click Add Item
  • Click the drop-down for Select Alerts and review the options, then select Any
  • Ensure that the Action field displays Send, then click Apply
    Use the Do Not Send feature to create an alert in the logs, but not send any notification.
  • Click Apply again
    If multiple Policy Rules exist, they can be re-ordered here to cause policies to trigger in a desired order.
  • Click Save and Exit
Step
Create Alert Receiver at Namespace Level
  • In upper-left corner click Select Service, then type (or click) Multi-Cloud App Connect
       
    Verify that your namespace is selected.
  • In the Manage section, hover over Alerts Management then select Alert Receivers
    Notice that an Alert Receiver already exists for studentX, using the "shared" namespace.
  • Click Add Alert Receiver
  • Name: studentX-ns-alert-receiver
  • In the Description field, type studentX namespace alert receiver
  • Click the drop-down for Receiver and select Email
  • Email: studentX@f5.com
  • Click Save and Exit
Step
Verify Email for Namespace-Level Alert Receiver
Even though both Alert Receivers use the same email address, both must be verified before F5XC will use them for alerting because F5XC confirms valid Alert Receivers based on verification of its receiving function, not on the email address itself.
  • Verify your window displays Multi-Cloud App Connect > Manage > Alerts Management > Alert Receivers
  • Click the 3 dots for studentX-ns-alert-receiver and select Verify Email
  • Click Send email
    Even though the same email address was previously verified, F5XC requires confirmation of receipt at the Alert Receiver level, not the email level.
  • In production, open the corresponding email inbox and copy the six digit verification code. In training, the Instructor will display these emails and provide the verification code
  • Click the 3 dots for studentX-ns-alert-receiver and select Enter verification code
  • Paste the six digit verification code. If applicable, be sure any leading zeros are pasted
Step
Test Namespace-Level Alert Receiver
  • Verify your window displays Multi-Cloud App Connect > Manage > Alerts Management > Alert Receivers
  • Click the 3 dots for studentX-ns-alert-receiver and select Send Test Alert
  • Click Send test alert
  • In production, open the corresponding email inbox to view Test alert. In training, the Instructor will display these emails
Step
Configure Namespace-Level Alert Policy
  • Verify your window displays Multi-Cloud App Connect > Manage > Alerts Management > Alert Receivers
  • In the Manage section, hover over Alerts Management then select Alert Policies
    Notice that the Alert Policy created earlier at the Tenant level for Namespace "shared" already exists.
  • Click Add Alert Policy
  • Name: studentX-ns-alert-policy
  • In the Description field, type studentX namespace alert policy
  • In the Alert Receiver Configuration section, click the drop-down for Alert Receivers and select studentX/studentX-ns-alert-receiver
    Note that the Tenant-Level Alert Receiver created earlier is available. This allows alerts across all namespaces to use the same Alert Receiver.
  • On the far-right side click the slider for Advanced Fields
  • In the Receiver Default Notification Parameters section, click Configure
  • Change value for Notify Interval For a Alert to 30m
    All fields must be in 2-digit number form (e.g. 60m, 60s)
  • In the Notification Grouping section, verify that the Wait to Notify value is 30s
  • In the Notification Grouping section, change Notify Interval for a Group to 30s
  • Click Apply
Step
Configure Namespace-Level Alert Policy Rules
  • Verify your window displays Multi-Cloud App Connect > Manage > Alerts Management > Alert Receivers > [Add Alert Policy] > Metadata
  • In the Policy Rules section, click Configure
  • Click Add Item
  • Click the drop-down for Select Alerts and review the options, then select Any
  • Ensure that the Action field displays Send, then click Apply
  • Click Apply again
  • Click Save and Exit
Notice that Alert Policies are displayed for both this namespace (studentX) and for the entire Tenant (shared).
Step
Create Active Alert Policies
  • Verify your window displays Multi-Cloud App Connect > Manage > Alerts Management > Alert Policies
  • In the Manage section, hover over Alerts Management, then select Active Alert Policies
  • Click Select Active Alert Policies
  • Click Add Item
  • Click the drop-down for Alert Policies
    Notice that all shared Alert Policies from all students are displayed, along with the namespace level Alert that you created.
  • Select shared/studentX-tenant-alert-policy
  • Click Save and Exit
Step
View Tenant-Level Alert Policies
  • In upper-left corner click Select Service, then type (or click) Shared Configuration
  • In the Manage section, hover over Alerts Management and explore the options
    • Notice there is not a section named "Active Alert Policies"
    • Only Tenant-Level Policies and Receivers are displayed
Step
Trigger Tenant-level alert
The easiest way to trigger a Tenant-level alert is to create or delete a user account (Administration > IAM > Users). This only needs to be done once; the action will be demonstrated by the instructor to prevent accidental user deletion in the console.
Step
View Tenant-level Alert Notifications
  • In upper-left corner click Select service, then type (or click) Audit Logs & Alerts
  • In the Notifications section, click Alerts
  • Click All Alerts tab
    • Change the Refresh rate as needed to display all historical alerts
    • It is likely that not many alerts are displayed since site setup is so recent
  • Click the Active Alerts tab
  • Refresh the screen as needed
       
    Displayed is a "Major" alert in Group "UAM" for "User Created."
Step
View Namespace-level Alerts in F5XC Console
  • In upper-left corner click Select Service, then type (or click) Multi-Cloud App Connect
      
    Verify that your namespace is selected.
  • In the Notifications section click Alerts
  • View the All Alerts and Active Alert tabs and refresh the window and refresh rate
    • Notice that the User Created Alert is not displayed because it occurred at the Tenant level
    • Other namespace-level alerts may appear here, but likely not many since site setup is so recent
Step
View alerts in email inbox
   
Instructor will display email alerts.
Q: Why is there not an email alert for the created User?
A: Because tenant-level alerts cannot be triggered by namespace-level alert policies
  
Feature or Bug? Either way be aware that even if a tenant-level alert is "active" within a namespace's "Active Alert Policies," tenant-level activities that should seemingly generate an alert cannot be triggered by namespace-level alert policies.
Step
View Tenant-level Alerts and generate notification
  • In upper-left corner click Select service, then type (or click) Audit Logs & Alerts
  • In the Alerts Management section, explore Alert Policies and Alert Receivers
    • Alert Policies display only tenant-level alert policies (and likely a client-side defense policy)
    • Alert Receivers display only tenant-level alert receivers (and likely a client-side defense receiver)
  • In the Alerts Management section, select Active Alert Policies
  • Click Select Active Alert Policies
  • Click Add Item
  • Select shared/studentX-tenant-alert-policy
  • Click Save and Exit
Step
Trigger Tenant-level alert
The easiest way to trigger a Tenant-level alert is to delete the user account created previously. The instructor will perform this action.
Step
View alerts in email inbox
   
Instructor will display email alerts.
Step
Cleanup
To prevent inbox overload, remove active Alert Policies
  
  • In upper-left corner click Select service, then type (or click) Audit Logs & Alerts
  • In the Alerts Management section, click Active Alert Policies
  • Click Select Active Alert Policies
  • For all Alert Policies, click the 3 dots and select Delete
  • Click Save and Exit

Expected Results

Students have:
  • Deployed Alert Policies and Alert Receivers at the Tenant and Namespace level
  • Viewed Alerts in the F5XC console
  • Viewed Alert information in email
        
      

Review

  • Alerts are generated throughout class so view these alerts periodically on the Audit Logs & Alerts page for Tenant-level alerts, and the Alert page for each Load Balancer
  • Ensure that Alert Intervals are set to two digits, otherwise no alerts will be triggered. There is otherwise no error message
  • The Instructor triggers tenant-level alerts by creating and then deleting user account(s)
  • Email is used as the alert receiver in class, but Slack, PagerDuty, OpsGenie, and SMS are also available options
